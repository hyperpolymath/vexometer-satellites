= Contributing to Vexometer Satellites
:toc:

== Overview

This umbrella repository coordinates the vexometer satellite ecosystem. Contributions fall into three categories:

1. **Documentation improvements** (this repo)
2. **Protocol enhancements** (this repo)
3. **Satellite implementations** (individual satellite repos)

== Contributing to This Repository

=== What to Contribute

[cols="2,3,2", options="header"]
|===
|Category |Examples |Review Required

|Documentation
|Fix typos, clarify protocols, improve examples
|Standard PR review

|Satellite Registry
|Add new satellite, update status, fix links
|Maintainer approval required

|Protocol Changes
|Modify vexometer-trace-v1 or efficacy-v1 schemas
|Requires vexometer team approval

|Tooling
|Validation scripts, CI improvements, automation
|Standard PR review
|===

=== How to Contribute

1. **Fork** this repository
2. **Create a branch**: `git checkout -b improve-docs`
3. **Make changes**:
   - Add SPDX headers: `// SPDX-License-Identifier: AGPL-3.0-or-later`
   - Use AsciiDoc for documentation
   - Validate JSON schemas if modifying protocols
4. **Test locally**:
+
[source,bash]
----
just check  # Runs linters and validators
----
5. **Submit PR** with clear description

=== Code Standards

* **AsciiDoc**: Use for all documentation (not Markdown)
* **SPDX Headers**: All files must have license headers
* **Links**: Verify all external links work
* **Status Accuracy**: Satellite registry must reflect actual repo state

=== Commit Messages

Use conventional commits:

[source]
----
docs: fix typo in integration protocol
feat: add vex-lazy-eliminator to registry
chore: update satellite status badges
----

== Protocol Change Process

Changes to integration protocols (vexometer-trace-v1, vexometer-efficacy-v1) require:

1. **Discussion**: Open issue describing proposed change
2. **Consensus**: Agreement from vexometer maintainers + â‰¥2 satellite maintainers
3. **Version bump**: Create new protocol version (don't break existing)
4. **Documentation**: Update specs in docs/
5. **Implementation**: Update vexometer validation tooling
6. **Migration period**: Support old version for â‰¥6 months

== Review Process

=== Documentation PRs

* Reviewed by any maintainer
* Merged within 1 week
* Focus: accuracy, clarity, links work

=== Satellite Registry PRs

* Reviewed by maintainer
* Verification required:
  - Repository exists
  - Meets satellite requirements
  - CI passing
  - SECURITY.md present
* Merged within 2 weeks

=== Protocol Change PRs

* Reviewed by vexometer team + satellite maintainers
* Requires consensus (may take 1+ months)
* Backwards compatibility required
